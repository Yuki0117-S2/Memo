Anima LoRA Training Tips

모든 값에 근거 등급이 붙어 있어. 사실은 공식 문서·개발자 발언, 경험칙은 커뮤니티 후기(데이터셋 따라 흔들림), 불확실은 교차 검증된 합의 없음 — 직접 실험해서 채울 것. 수치는 출발점이지 정답이 아님.

적은 데이터셋 전략
repeat로 총 스텝 확보 경험칙
이미지가 적으면 repeat를 올려서 총 스텝(이미지 수 × repeat × epoch ÷ batch)을 목표 구간에 맞춘다. 20~30장이어도 커버 가능.
장수보다 화풍 일관성 경험칙
화풍이 흔들리는 50장보다 딱 떨어지는 20장이 낫다. 작가 초기작·최근작을 섞으면 흐릿한 평균이 나온다. 고해상도 원본만 추릴 것.
중간 체크포인트 필수 경험칙
데이터가 적을수록 과적합이 빨리 온다. epoch마다 저장하고 프리뷰를 비교해서, 원본을 베끼기 시작하기 직전 체크포인트를 고른다.
캡션 전략
화풍 요소는 태그에서 뺀다 불확실
색감·선 굵기 같은 화풍 요소는 캡션에 넣지 않고, 내용물(1girl, 배경 등)만 태깅한다. 태그 안 된 요소가 로라에 흡수되는 원리. SD 계열 통설 · Anima에서의 검증은 확인 불가.
트리거 워드는 한 곳에만 사실
TrainFlow 기준, GUI의 Trigger Word 칸 또는 캡션 파일 중 한 곳에만 넣는다. 양쪽에 넣으면 중복되어 품질이 떨어질 수 있다.
shuffle_caption + keep_tokens 경험칙
태그 순서를 섞어 특정 태그 의존을 줄인다. 트리거 워드를 쓴다면 keep_tokens=1로 맨 앞 고정. SD 계열 통설.

어느 파일이 진짜 작동하냐 사실
src/nodes.py 만 실제로 로드됨 (__init__.py가 from .src.nodes로 불러옴). 폴더 밖에 있는 nodes.py는 import 안 되는 유령 파일이니 무시. 수정은 반드시 ...\comfyui-instant-lora\src\nodes.py 에 할 것.
learning_rate 자릿수 복구 (업데이트 때마다 초기화됨) 사실
src/nodes.py에서 learning_rate_override 줄을 검색해서(행 번호는 버전마다 다름) 아래로 통째 교체. step 0.0001 → step/round 0.000001이면 6자리까지 안 뭉개짐.
"learning_rate_override": ("FLOAT", {"default": 0.0, "min": 0.0, "max": 1.0, "step": 0.000001, "round": 0.000001}),
노드 세대 구분 사실
태깅 노드가 두 갈래로 존재함. 자동 태거 버전(general_threshold·character_threshold 위젯 있음 = WD14 자동 태깅)과 수동 버전(tags 박스만 있음). 지금 쓰는 건 자동 태거 버전 — 캡션 없어도 WD14가 알아서 생성.
수정 후 체크리스트 경험칙
① ComfyUI 재시작 → 콘솔에 빨간 에러 없으면 로드 성공. ② Reference Train Options 노드를 새로 꺼내기 (기존 노드는 워크플로에 옛 정의가 박제돼 있어 반영 안 됨). ③ lr 칸에 0.00005 쳐서 6자리 유지되면 성공.
버전 주의 — preview(1~3)로 구운 로라는 Anima 1.0에서 "소프트하게" 먹힌다. 1.0 이후로는 1.0 기준으로 새로 굽는 것을 권장. 향후 2.0, 3.0이 나오면 구버전 로라는 점점 더 어긋난다.
